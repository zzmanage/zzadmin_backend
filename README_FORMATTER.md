# 代码格式化工具使用说明

## 工具介绍

这个项目包含一个名为`format_code.py`的Python脚本，用于自动格式化项目中的所有Python代码。该工具集成了多种流行的代码格式化和检查工具，可以帮助团队保持一致的代码风格。

## 功能特点

1. **自动安装依赖**：脚本会自动检查并安装必要的代码格式化工具
2. **递归查找Python文件**：自动递归查找项目中的所有Python文件
3. **多工具协同**：使用以下工具进行代码格式化和检查：
   - **isort**：整理Python导入语句
   - **black**：代码格式化工具，强制一致的代码风格
   - **autopep8**：根据PEP 8规范格式化代码
   - **flake8**：代码质量检查工具
4. **智能排除**：自动排除不需要格式化的目录和文件（如虚拟环境、迁移文件等）

## 安装和使用

### 前提条件

- 已安装Python 3.6或更高版本
- 已激活项目的虚拟环境（推荐）

### 使用方法

1. 确保您在项目根目录下
2. 运行以下命令：

```bash
# Windows系统
python format_code.py

# macOS/Linux系统
python3 format_code.py
```

## 自定义配置

如果需要自定义格式化行为，可以修改`format_code.py`文件中的以下变量：

- `REQUIRED_PACKAGES`：需要安装的格式化工具列表
- `EXCLUDE_DIRS`：需要排除的目录列表
- `EXCLUDE_FILES`：需要排除的文件列表
- `format_with_autopep8`函数中的参数：可以调整autopep8的格式化选项
- `lint_with_flake8`函数中的flake8配置：可以调整flake8的检查规则

## 注意事项

1. **备份代码**：在首次使用格式化工具前，建议先备份您的代码或确保所有更改已提交到版本控制系统
2. **IDE集成**：许多现代IDE（如PyCharm、VS Code等）也提供了代码格式化功能，您可以将其配置为使用与本脚本相同的工具和规则
3. **CI/CD集成**：您可以将此脚本集成到CI/CD流程中，确保所有提交的代码都符合项目的代码规范

## 常见问题

### 1. 运行脚本时出现权限错误

**解决方法**：以管理员权限运行命令提示符或终端，或者修改脚本文件的权限。

### 2. 格式化后的代码与团队规范不一致

**解决方法**：修改`format_code.py`文件中的配置选项，特别是`autopep8`和`flake8`的参数，以匹配团队的代码规范。

### 3. 格式化过程中出现内存不足的错误

**解决方法**：对于大型项目，可以修改脚本以分批处理Python文件，而不是一次性处理所有文件。
